# hermes-a2a developer guide

## Source of truth

M7 pins upstream A2A v1.0.0 from `a2aproject/A2A` tag `v1.0.0` / commit `173695755607e884aa9acf8ce4feed90e32727a1`.

Pinned artifacts live under `spec/upstream/`:

- `SOURCE.json`
- `source-bundle-manifest.json`
- `a2a.proto`
- `specification.md`
- upstream license/config/docs snapshots
- `a2a-sdk-pypi-1.0.0.json`

The published v1.0.0 spec Section 1.4 states that `spec/a2a.proto` is the single authoritative normative definition of protocol data objects and request/response messages. This package uses `a2a-sdk==1.0.0` for generated proto/types, server routes, client/gRPC stubs, request handlers, and task stores.

## Package layout

```text
src/hermes_a2a/
  agent_card.py       # public-safe Agent Card construction
  app.py              # Starlette route factory using SDK JSON-RPC/REST routes
  config.py           # typed M17b instances.yaml loader and validation receipt writer
  conformance.py      # label/matrix gates
  executor.py         # synthetic local executor with receipts/projection
  grpc_server.py      # loopback-only gRPC context manager
  handler.py          # request handler policy wrappers
  live_executor.py    # gated Hermes profile executor with private stdout/stderr receipts
  policy.py           # local default-deny helpers
  projection.py       # peer-visible leak scanner
  receipts.py         # atomic private receipt writer
  serve.py            # sidecar runtime for validated instance configs
  service_runner.py   # systemd-friendly sidecar process entry point
```

## TDD and validation

Primary test command:

```bash
uv run --extra dev python -m pytest tests -q
```

Conformance receipt command:

```bash
uv run --extra dev python scripts/validate_local_conformance.py
```

M17b sidecar-triad receipt command:

```bash
uv run --extra dev python scripts/run_m17b_triad_pilot.py --overwrite-config
```

Gated M17c/M17d/M17e receipt commands require management-workspace approval receipts. Set `HERMES_A2A_MANAGEMENT_ROOT` or pass `--management-root` when the management workspace is outside the repository:

```bash
uv run --extra dev python scripts/run_m17c_live_executor_pilot.py --approval-receipt <management-root>/milestones/m17c/<approval>.yaml --profile default
uv run --extra dev python scripts/run_m17d_service_rollout.py --approval-receipt <management-root>/milestones/m17d/<approval>.yaml
uv run --extra dev python scripts/run_m17e_lan_pilot.py --approval-receipt <management-root>/milestones/m17e/<approval>.yaml --host <approved-lan-ip> --negative-ssh-host <user@unlisted-lan-host>
```

Clean-temp validation should exclude `.venv`, caches, and generated bytecode, then run the same commands in the copy.

## Binding decisions

- JSON-RPC, REST, and gRPC are implemented through the pinned SDK route/handler/stub layer.
- `SubscribeToTask` has a v1.0.0 prose/proto REST discrepancy: prose says POST, proto says GET. The SDK accepts both; proto/GET remains the canonical source while compatibility behavior is retained.
- HTTP protocol routes require `A2A-Version: 1.0`.
- gRPC smokes are loopback-only through `LocalGrpcServer`; M17b can also bind fixed loopback gRPC ports from a validated per-run roster.

## M17b config and sidecar runner

- `hermes_a2a.config` exposes typed dataclasses for the schema-version-1 roster and rejects invalid configs before any bind.
- Config path resolution is explicit `--config`, then `HERMES_A2A_INSTANCES`; missing or invalid paths fail closed.
- M17b validation is strict: exactly the three synthetic conceptual ids, `config_status: validated`, `live_execution_enabled: false`, `host_inventory_status: not_checked`, `auth.mode: test_ephemeral`, `127.0.0.1` only, distinct HTTP/gRPC ports, and receipt roots under `<management-root>/milestones/m17b/runs/<run_id>/receipts`.
- `hermes_a2a.serve.SidecarRuntime` starts foreground loopback-only HTTP and gRPC sidecars from one validated instance row and never invokes a live Hermes profile.
- `scripts/run_m17b_triad_pilot.py` mints or accepts a run id, writes the management roster, writes validation/run/bind/non-action receipts, exercises JSON-RPC/REST/gRPC, checks allowed/disallowed peers with in-memory `test_ephemeral` auth, verifies teardown, and writes an artifact manifest.

## Gated live executor, service runner, and LAN pilot

- `hermes_a2a.live_executor.HermesProfileExecutor` is opt-in only. It launches the approved Hermes profile with a proven command shape, bounded timeout/output limits, and private invocation receipts. Peer-visible A2A output uses the same receipt/projection gates as the synthetic executor.
- `hermes_a2a.serve.SidecarRuntime` remains synthetic by default; `--executor live` requires `--live-profile` and is only for approved pilots.
- `hermes_a2a.service_runner` is the stable entry point used by M17d user-level systemd units. It accepts explicit HTTP/gRPC ports, receipt root, test-token env/file, executor mode, and absolute `--hermes-command` for user-service PATH isolation.
- `scripts/run_m17d_service_rollout.py` writes finite unit files, reloads/restarts only those units, waits for selected ports, captures `systemctl --user`/PID/`ss`/log/readiness evidence, smokes HTTP+gRPC, and records rollback commands.
- `scripts/run_m17e_lan_pilot.py` is synthetic-only unless a separate live-over-LAN gate exists. It binds one foreground HTTP sidecar to an exact named LAN address, can run a quoted remote SSH TCP/HTTP negative probe from an approved unlisted host, writes `blocked` instead of `passed` when negative unlisted-host reachability evidence is missing or inconclusive, and tears the listener down before exit.

## Safety wrappers

The local Hermes safety layer is not a protocol fork. It wraps SDK execution/egress:

- `projection.assert_safe_peer_visible` blocks private paths, MCP/tool surface strings, secret-shaped tokens, and environment assignments.
- `ReceiptStore` writes a private JSON receipt atomically before peer-visible completion status is enqueued.
- `SafeRequestHandler` denies non-loopback push URLs by default.
- Extended Agent Card access can be protected by `x-hermes-a2a-key` in local tests.
- `scan_forbidden_labels` blocks premature conformance labels unless matrix evidence exists.
